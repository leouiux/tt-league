01
System Overview

시스템 개요

용문 탁구 리그 운영에 특화된 풀스택 웹 애플리케이션으로, 리그전 조 편성부터 토너먼트 우승자 결정까지 전 과정을 단일 인터페이스에서 처리합니다.

03
Data Model

데이터 모델

모든 데이터는 단일 JSON 객체(masterData)에 타임스탬프 ID 키로 관리됩니다.

// masterData 최상위 구조 masterData = { "20260315_143022": // ID = YYYYMMDD_HHMMSS (저장 시각) { id: "20260315_143022", date: "2026-03-15", title: "용문리그 - 용문부", targetWins: 2, // 경기 방식 (2선승 or 3선승) eliminateCount: 1, // 조별 탈락 인원 savedAt: "2026. 3. 15. 오후 2:30:22", groups: { "1조": { names: ["홍길동", "이순신", ...], grades: { "홍길동": 2, "이순신": 5 }, playerIds: { "홍길동": 1 }, // 번호 results: { "홍길동": { "이순신": { s1:2, s2:1, done:true } } }, manualRanks: { "홍길동": 1 }, // 동률 수동 순위 manualOrder: ["홍길동", ...], // 드래그 순위 modifiedNames: { "이영희": true } // 명단수정 하이라이트 } }, tournament: { seeds: [{ name, group, rank, grade, seed, isEliminated, isVacant }], rounds: [{ name:"결승", matches:[{ id, player1, player2, winner, completed, isWalkover }] }], champion: "홍길동" } } }

F03
Feature

리그전 대진표

📊 매트릭스 구조
  • N×N 교차 매트릭스 (자기 자신 제외 — 셀로 표시)
  • 좌측 열: 이름 입력 input (편집 가능)
  • 상단 행: 이름 th (좌측 입력과 실시간 동기화)
  • 각 셀: 점수 select (0:0 ~ 선승:0 범위)
  • 이긴 셀 배경: 연두색 하이라이트
✏️ 이름 직접 수정
  • 클릭 시 파란 테두리 2px 편집 모드
  • 입력 형식: 홍길동9 (이름+등급 붙여쓰기)
  • blur 시 전체 데이터 키 일괄 교체
  • 공백 또는 "-" 입력 → 공석 처리
  • 수정 시 순위표·인쇄·캡쳐 자동 반영
🔢 점수 입력 연동 처리
한쪽 셀에 점수 입력 시 → 대칭 셀(상대방 점수) 자동 역산 반영.
예: 홍길동 vs 이순신 = 2:1 입력 → 이순신 vs 홍길동 셀에 1:2 자동 입력.
점수 수정 시 → 드래그 수동 순위(manualOrder, manualRanks) 자동 초기화.

공석 처리

🔲 공석 발생 경로
  • 이름 input에 공백 또는 "-" 입력 후 blur

영역공석 처리 방식
순위표공석은 항상 최하위. 2개 이상이면 순서대로 하위 순위 배정

토너먼트 (공석 x개 ≥ 탈락 n)공석만 탈락, 나머지 진출
토너먼트 (공석 x개 < 탈락 n)공석 전원 탈락 + 비공석 하위 (n-x)명 탈락
토너먼트 브라켓 (공석 2개 이상)첫 번째 공석 탈락 · 나머지는 진출, 상대방 자동 부전승
전체 명단 인쇄·캡쳐"-"으로 표시

밴드에 올릴 최종명단 캡쳐PNG 다운로드html2canvas 라이브러리. YYYY.MM 날짜 h1에 자동 표기. 파일명에 날짜 포함
⚠️
인쇄 시 브라우저 설정에서 머리글/바닥글 체크해제, 배경그래픽 체크 권장. 안내 문구 인쇄 화면에 표시됨.
F09
Feature

데이터 저장·불러오기

💾 저장 방식
  • 저장 클릭 시 현재 대회 데이터를 새 타임스탬프 ID로 복사 → 스냅샷 생성
  • 기존 ID의 데이터는 그대로 유지 → 버전 이력 보존
  • 서버에 POST /api/save로 전체 masterData 전송
  • 저장 성공 시 alert로 저장 시각 안내
📂 JSON 내보내기·가져오기
  • 내보내기: 전체 masterData를 JSON 파일로 다운로드
  • 파일명: 용문리그_YYYYMMDD_HHMMSS.json
  • 가져오기: JSON 파일 선택 → 기존 데이터와 병합 (덮어쓰기 아님)
  • 서버 자동 저장 후 선택기 갱신
🚨
Railway 주의: git push 시 컨테이너 재생성 → data/league_db.json 초기화 위험. Railway Volume 마운트(/app/data) 설정 필수. DATA_DIR 환경변수로 경로 지정.

Cautions

주의사항

🚨
데이터 유실 위험
수동 저장 없이 이탈 시 변경사항 소실. 점수 입력·이름 수정 후 반드시 💾 저장 클릭.
🚨
Railway Volume 미설정
git push 시마다 league_db.json 초기화. Volume 마운트(/app/data) + DATA_DIR 환경변수 필수.
⚠️
토너먼트 재생성 불가 복구
토너먼트를 재생성하면 기존 결과 삭제. confirm 전에 반드시 저장.
⚠️
공석 처리 후 저장 필수
명단 수정으로 발생한 공석은 저장하지 않으면 다음 접속 시 복원되지 않음.
✅ 운영 체크리스트
  • Railway Volume 마운트 확인 (/app/data)
  • DATA_DIR 환경변수 설정 확인
  • 경기 결과 입력 완료 후 즉시 저장
  • 명단 수정 후 저장 확인
  • 토너먼트 진행 전 최종 저장
  • 인쇄 시 브라우저 인쇄 설정(머리글/바닥글 해제, 배경그래픽 체크) 안내
  • 서버 재배포(git push) 전 데이터 JSON 내보내기 백업
